# Change: 新增进程管理（内存加速）能力

## Why
用户在日常使用中经常感觉系统卡顿，希望像厂商工具那样提供“扫描 → 选择 → 结束后台进程”的加速能力。Linux 的内存会被用作文件缓存，单纯“清理垃圾/清理缓存”并不能直接提升流畅度；更有效的方式通常是识别并结束异常占用资源（尤其是内存/Swap 压力下）的后台进程，同时维持本项目“默认只读、写操作需显式确认”的安全边界。

## What Changes
- 新增能力 **process-manager**：提供一个进程管理页面，展示系统内存/Swap 概览与进程列表（按内存占用排序），支持搜索与多选。
- 提供“结束进程”操作：默认发送 SIGTERM；若目标在短时间内仍存活，允许用户选择“强制结束”（SIGKILL）。
- 安全策略：
  - 仅允许结束当前用户拥有（同 UID）的进程；对 root/其他用户进程不提供直接结束（按钮禁用并提示需要管理员权限）。
  - 所有结束操作必须二次确认，并明确提示可能导致未保存数据丢失。
  - 禁止结束应用自身进程（packlens）。
- UX：扫描与列表渲染需保持 UI 可响应；结束后自动刷新列表并提示成功/失败结果。

## Impact
- Affected specs: process-manager（新增）
- Affected code（预计）: `src/services/`（新增进程扫描与结束服务），`src/ui/`（新增页面），`src/window.rs`（接入侧边栏导航与 i18n）
- 权限：读取 `/proc` 无需 root；结束进程使用 `kill(2)`，默认仅对同 UID 进程执行，无需 sudo
